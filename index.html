What is HTML

HTML (HyperText Markup Language) is like the instruction manual for building a website, where each element is a unique Lego piece with a role in the structure. Just like a Lego set uses blocks of different shapes and colors to build castles, spaceships, or cities, HTML uses tags to construct and organize content on a webpage.

HTML Structure

Imagine this as the foundation or the baseplate for a Lego set. Every HTML page starts with this basic structure:

<!DOCTYPE html>
<html>
  <head>
    <title>My Web Page</title>
  </head>
  <body>
    <!-- Your content goes here -->
  </body>
</html>

Each part is like a section of your Lego project:

  • <!DOCTYPE html> is like the set number on a Lego box. It tells the browser (or "builder") which version of HTML it’s working with.
  • <html> is the overall container, like the base box holding all the Lego pieces together.
  • <head> is similar to the instruction booklet, providing extra details and instructions like the page's title that you might not see directly.
  • <body> is the main area where all the visible "Lego bricks" go.
Elements and Tags

Elements are like Lego bricks, and tags are the shapes that determine each brick’s role. To create a paragraph, you use the <p> tag, much like how you’d pick a specific brick for a wall in a Lego Harry Potter castle:

<p>This is a paragraph.</p>

In this instance, the tags <p> and </p> make up a singular Lego brick that tells the browser this brick is a paragraph.

HTML Titles and Headings

Headings are like signs in a Lego set that label each room or area. Imagine you’re building Hogwarts with a big banner at the top labeled "Great Hall." HTML headings range from <h1> (a large banner like “Hogwarts”) to <h6> (smaller labels, like “Potions Classroom”). They organize the sections of your page:

<h1>Main Title</h1>
<h2>Subsection Title</h2>
Paragraphs and Text Blocks

A paragraph groups together a chunk of text, like building separate rooms or areas in the Lego Star Wars Millennium Falcon set. Using the <p> tag keeps ideas organized and distinct, just like how each section of the Falcon has its own purpose—such as the cockpit, crew quarters, and main hold—each adding to the overall structure of the ship:

<p>This is a paragraph of text that helps keep information and ideas separated.</p>
Links and Navigation

Links are like secret doors between Lego buildings! Just as you can help your troops of Lego Knights invade a fort of Orcs and Goblins with a bridge, links can bring your users to different pages. The <a> tag with an href attribute is like a bridge that takes you somewhere new:

<a href="https://www.example-link.com">Click here to visit Example</a>

This creates a link that says "Click here to visit Example" that users can click to go to your desired location.

Adding Images

Images are like stickers you can put on your Lego build. You use the <img> tag to "stick" an image onto your page. The src attribute is where the sticker originates from, and alt is like a description of the image:

<img src="lego-car.jpg" alt="A nice lego car">

Using alt text is like adding a label to your stickers, helping others understand the picture if it doesn't show up.

Lists: Ordered and Unordered

Lists are like stacks of Lego pieces organised in order. You can make:

  • Unordered lists (<ul>) that use bullet points.
  • Ordered lists (<ol>) that use numbers.

Each list item is like a separate brick that stacks in a line:

<ul>
  <li>Item 1</li>
  <li>Item 2</li>
  <li>Item 3</li>
  <li>Item 4</li>
</ul>
HTML Attributes

Attributes are like the special features or stickers you can add to a Lego brick. They give extra details about an element. A few examples are as follows:

  • href tells a link where to go.
  • src tells an image where the picture is stored.
  • alt describes the image.

Think of these as “power-ups” that add functionality or instructions to each piece, similar to adding a propeller on a Lego City helicopter that actually spins.

Formatting Text in HTML

Similar to how you can customize Lego figures to have different clothes and accessories, HTML tags can change how text appears:

  • Bold (<b> or <strong>) makes text stand out.
  • Italic (<i> or <em>) emphasizes text.
  • Underline (<u>) puts a line under text.

Example:

<p>This is <b>bold</b>, <i>italic</i>, and <u>underlined</u> text.</p>

The example code will read as "This is bold, italic, and underlined text."

Note however, text formatting is more typically handled in Cascading Style Sheets (CSS) rather than HTML.

Tables for Structured Data

Tables are important for organizing information in rows and columns. The following elements are applicable when working with tables:

  • <table> starts the table structure.
  • <tr> adds a new row.
  • <th> adds a header cell (label).
  • <td> adds a a regular data cell.

Example:

<table>
  <tr>
    <th>Fruit</th>
    <th>Color</th>
  </tr>
  <tr>
    <td>Apple</td>
    <td>Red</td>
  </tr>
</table>

The same way you would sort tools in a Lego workshop, tables serve as an easy way to organise data.

Forms and Input Fields

To gather user opinions on a particular subject you will need to use forms to collect this data, like how Google Reviews have enabled enjoyers of The Lego Batman Movie to report a rating of 4.8 stars out of 5. A form can include:

  • </form> wraps all of a form's elements.
  • </input> creates single-line text boxes for a user to write in.
  • </textarea> creates larger boxes when longer text may be required from a user.

Example:

<form>
  <label for="name">Name:</label>
  <input type="text" id="name" name="name">
</form>
Semantic HTML

Semantic HTML is like using different colors to make Lego city sections easily identifiable - red for fire stations, blue for police, etc. Using tags as listed below helps organise the page:

  • <header> represents the introductory section of a webpage. It typically contains important information such as the title, logo, navigation links, or introductory content relevant to that section.
  • <footer> is found at the bottom of a webpage. It usually includes concluding information, like copyright details, contact information, or links to related resources. It provides closure and additional reference material for the user.
  • <section> elements are used to group related content together within a webpage. Each section represents a distinct thematic grouping, helping to organize content logically. This makes it easier for both users and search engines to understand and navigate the page’s structure.
  • <article> elements are used for self-contained content that could stand on its own, like a news article, blog post, or product description.

Using Semantic HTML makes your page easier to navigate and understand.

Why HTML Matters

HTML is the foundation of every webpage. Understanding HTML allows you to structure and organize content effectively, making it accessible and readable for both users and search engines. Mastering HTML is essential for anyone interested in web development, as it provides the basic building blocks that allow for styling with CSS and adding interactivity with JavaScript. With a solid grasp of HTML, you gain the tools to create clear, functional web pages that service as the core of the modern internet.
